Cara Isi Deposit

Berikut ini adalah format transaksi Pulsa yang telah kami kami sediakan dan semua format transaksi di bawah ini harap di kirim ke SMS CENTER, TELEGRAM CENTER, NIMBUZZ CENTER, YM CENTER atau pun GTALK CENTER kami.

ISI PULSA TUNGGAL


Isi pulsa tanpa kode
Format : Nominal.No HP.Pin
Contoh : 5.08123456789.1234
Isi Pulsa pakai kode ( khusus transaksi I-SAT SMS dan GPRS
Format : Kode.NO HP.PIN
Contoh : IS20.0856567890.1234

ISI PULSA LISTRIK PRABAYAR/TOKEN PLN


Format : Kode.No Kartu/Meteran.PIN
Contoh : PLN20.11223344556.1234

MULTI TRANSAKSI


Format : Nominal.NO HP.Nominal.NO HP.PIN
Contoh : 20.081234567890.5.0851234567890.1234

ISI PULSA KE NOMER DAN PRODUK SAMA DLM 1 HARI


Format : Nominal.2.NO HP.PIN
Contoh : 20.2.081234567890.1234

CEK SALDO DEPOSIT


Format : S.PIN
Contoh : S.1234

MEMINTA REKAP TRX


Format : CEK.PIN
Contoh : CEK.1234

CEK TRANSAKSI PER TANGGAL


Format : CEK.TANGGAL.PIN
Contoh : CEK.12.1234

CEK STATUS TRANSAKSI


Format : CEK.TUJUAN.PIN
Contoh : CEK.08234567890.1234

CEK MUTASI SALDO


Format : LAP.PIN
Contoh : LAP.1234

CEK LAPORAN MUTASI SALDO PER TANGGAL


Format : LAP.TANGGAL.PIN
Contoh : LAP.12.1234

GANTI PIN


Format : GP.PIN Baru.PIN Lama
Contoh : GP.1234.1111

UBAH NAMA


Format : GN.NAMA BARU.PIN
Contoh : GN.PUTUTCELL.1234

PARALEL NO HP TANSAKSI


Format : TAMBAHHP.NOHP.PIN
Contoh : TAMBAHHP.081234567890.1234

HAPUS PARALEL NO HP TANSAKSI


Format : HAPUSHP.NOHP.PIN
Contoh : HAPUSHP.081234567890.1234

PARAREL YM TANSAKSI (Pararel YM Hanya dilayani dari No HP Utama Anda)


Format : TAMBAHYM.EMAILYAHOO.PIN
Contoh : TAMBAHYM.oke_pulsa@yahoo.co.id.1234

HAPUS YM TANSAKSI


Format : HAPUSYM.EMAILYAHOO.PIN
Contoh : HAPUSYM.oke_pulsa@yahoo.co.id.1234

PARAREL GTALK TANSAKSI


Format : TAMBAHGT.IDGTALK.PIN
Contoh : TAMBAHGT.oke_pulsa@gmail.com.1234

HAPUS GTALK TANSAKSI


Format : HAPUSGT.IDGTALK.PIN
Contoh : HAPUSGT.oke_pulsa@gmail.com.1234

LIHAT DAFTAR PENGIRIM SEMUA TIPE


Format : CEKHP.PIN
Contoh : CEKHP.1234

LIHAT DAFTAR PENGIRIM TIPE SMS UNTUK DOWNLINE


Format : CEKHP.KODE AGEN.PIN
Contoh : CEKHP.IR01234.1234

KONFIRMASI DEPOSIT


Format : KOM.isi konfirmasi
Contoh : KOM.Sudah transfer Rp.100.923 via BRI a/n Dian saldo belum masuk mohon dicek.
*NB: Deposit input otomatis. Konfirmasi deposit hanya dilakukan setelah setengah jam saldo belum masuk.

KOMPLAIN TRANSAKSI


Format : KOM.ISI KOMPLAIN
Contoh : KOM.trx 5.081998495777 tgl 10/02/2015 pulsa belum masuk, tapi saldo sudah kepotong mohon dicek

MENDAFTARKAN AGEN / DOWNLINE BARU


Format : Daftar#nama#Kota#NO HP#MARKUP#PIN
Contoh : DAFTAR#PULSA# WONOGIRI#085247034272#50#1234

TRANSFER SALDO ANTAR ID AGEN


Format : TD.ID AGEN.NOMINAL.PIN
Contoh : TD.IR00123.50000.1234

UBAH MARKUP DOWNLINE


Format : UP.KODEAGEN.MARKUP.PIN
Contoh : UP.IR01234.100.1234

CEK SALDO DOWNLINE


Format : S.KODE AGEN.PIN
Contoh : S.IR01234.1234

CEK SMS CENTER AKTIF


Format : CENTER
Contoh : CENTER

RESET PIN MEMBER


Format : PIN.KODE AGEN.PIN
Contoh : PIN.IR01234.1234

AKTIFKAN DOWNLINE/RESELLER


Format : AKTIF.KODE AGEN.PIN
Contoh : AKTIF.IR01234.1234

BLOKIR DOWNLINE/RESELLER


Format : BLOKIR.KODE AGEN.PIN
Contoh : BLOKIR.IR01234.1234

LIHAT DAFTAR DOWNLINE/RESELLER


Format : LDL.PIN
Contoh : LDL.1234

LIHAT DAFTAR DOWNLINE DARI DOWNLINE (SUB-DOWNLINE)


Format : LDL.KODE AGEN.PIN
Contoh : LDL.IR01234.1234

CEK HARGA UP TO DATE


Format : CH.KODE PRODUK.PIN
Contoh : CH.S.1234

SMS END USER / PESAN IKLAN


Format : IKLAN.ISI IKLAN
Contoh : IKLAN.Terimakasih telah membeli pulsa di AGUS CELL, kami juga melayani Pembelian Pulsa Listrik, Game dan PPOB.

PENTING : Sebelum anda melakukan pengiriman SMS transaksi anda, sebaiknya anda setting validitas pesan/SMS anda menjadi 1 jam. Hal ini bertujuan untuk menghindari pesan SMS pending yang anda kirim hari ini dan akhirnya terkirim besoknya/lain hari dan menyebabkan terjadinya transaksi ke nomer yang sama meski pada hari tersebut anda merasa tidak mengirimkan transaksi tersebut. Hal inilah yang tidak anda sadari/mengerti oleh semua pebisnis pulsa via SMS. Bagi Member yang tidak mengerti akan hal ini,mereka selalu menyalahkan server karena memproses transaksi tersebut. padahal nyata di server ada perintah transaksi itu tanpa kita tau itu pesan/sms transaksi kapan. karena server berjalan otomatis dan akan memproses semua perintah sms yang masuk.
